
Настройка подключения к VPN туннелям WireGuard, использующие современную криптографию.
Рекомендуется импортировать файлы конфигурации WireGuard (.conf), или скопировав их содержание использовать кнпку "Вставить". Подробнее о туннелях WireGuard: https://www.wireguard.com/

Общедоступная конечная точка (Endpoint):
IP адрес и порт VPN сервера доступа.

Закрытый ключ (Private key):
Закрытый ключ WireGuard для одного узла, предоставляется провайдером VPN, является уникальным.

Предварительный общий ключ (Shared key):
Предварительный общий ключ сервера WireGuard, предоставляется провайдером VPN, является уникальным.

Открытый ключ: (Public key):
Открытый ключ WireGuard для одного узла, предоставляется провайдером VPN, является уникальным.

IP адрес:
Адреса, к которым будет привязан клиент, IPv4 или IPv6.  Предоставляется провайдером VPN

Список разрешенных сетей для использования VPN (AllowedIPs):
Диапазон IP-адресов, трафик которых маршрутизируются в VPN тунель.

DNS
Сервер доменных имен, используемый для преобразования имен хостов в IP-адреса для VPN-клиентов.
Рекомендуется оставлять значение предоставленное провайдером VPN, это предотвратит утечку DNS-запросов за пределы VPN и раскрытие трафика. Утечки можно проверить с помощью http://dnsleak.com .

Период проверки VPN канала связи (Keepalive):
Отправлять периодические сообщения проверки активности, чтобы гарантировать рабочее соединение.
